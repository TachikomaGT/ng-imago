ng-imago Demo

Directive ng-imago

Simple use: set a set of images for different resolutions. The above image will load automatically.

Override default options: pass a JSON object ng-imago="{'portrait':true}" to override the default options of the module (see Defaults in app.js)

Sequential loading: setting the queue-index to a number will cause the image to be loaded after all the images having lower queue-index (or the ones without queue-index). The image above will be loaded after the previous two

Delayed loading: you can choose to not load the image automatically with auto-load="false". You will have to explicit change the attribute or load image via angular function (see Code -> Loading manually).

Since you should load the images manually, you can set a load-group="NAME_OF_THE_GROUP"attribute to load all the imges of that group. You should use either queue-index or queue-index.

You can load an image by the id. Notice that you can also pass the url within a ng template.

You can load an image by the queue-index. {{url_desktop}}
Notice that the image with queue-index="4" is not still loaded event tough it has auto-load='true'. Because the queue-index="3" is not still loaded. As you click the button above and the image with queue-index="3" will be loaded, also the image with queue-index="4" will be loaded.

You can load an image previously settes with auto-load="false" changing the attribute's value.
