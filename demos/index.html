ng-imago Demo

Responsive resources

Simple use: a set of images sources for different resolutions. The above image will load automatically.

<img  ng-imago 
            
            small="http://www.placehold.it/320x100&text=small, http://www.placehold.it/320x200&text=small-@2x" 
            medium="http://www.placehold.it/420x100&text=medium, http://www.placehold.it/420x200&text=medium-@2x" 
            large="http://www.placehold.it/620x100&text=large, http://www.placehold.it/420x200&text=large-@2x" 
            xlarge="http://www.placehold.it/720x100&text=xlarge,  http://www.placehold.it/720x200&text=xlarge-@2x"
            
            custom="http://www.placehold.it/5204x100&text=custom" />

Override default options: pass a JSON object to override the default options of the module (see Defaults).

    <img ng-imago="{'portrait':false, small:480, medium:700}"
            
            small="{{url_small}}" 
            medium="{{url_medium}}" 
            large="{{url_small}}" 
            xlarge="{{url_xlarge}}" />

Sequential loading

Sequential loading: setting the queue-index to a number will cause the image to be loaded after all the images having lower queue-index (or the ones without queue-index). The image above will be loaded after the previous two

Delayed loading: you can choose to not load the image automatically with auto-load="false". You will have to explicit change the attribute or load image via ngImagoService function (see Code -> Loading manually).

Since you should load the images manually, you can set a load-group="NAME_OF_THE_GROUP"attribute to load all the imges of that group.

You can load an image by the id. Notice that you can also pass the url within a ng template.

You can load an image by the queue-index. {{url_large}}
Notice that the image with queue-index="4" is not still loaded event tough it has auto-load='true'. Because the queue-index="3" is not still loaded. As you click the button above and the image with queue-index="3" will be loaded, also the image with queue-index="4" will be loaded.

You can load an image previously settes with auto-load="false" changing the auto-load attribute's value.

You can load an image by the class name.

Resizing

To resize the image to his parent, you can use another directive called "imago-resize" passing some options.
If the image has a position absolute or fixed, it will be moved with top an left, otherwise with 'margin-top' and 'margin-left'.
<img ng-imago medium="{{url_medium}}" auto-load="false" class="imag_to_resize" imago-resize="{scale:'fit', center:false, container:'window'}" />
